# КАК НАЙТИ ПРАВИЛЬНЫЙ АДРЕС СЕРВЕРА ДЛЯ SSH

## ПРОБЛЕМА

Домен `parser-auto.site-access.ru` не резолвится для SSH подключения.

Это нормально - домен может быть настроен только для веб-доступа, а SSH использует другой адрес.

---

## РЕШЕНИЕ 1: НАЙТИ IP АДРЕС В ПАНЕЛИ BEGET

1. Войдите в панель управления Beget
2. Найдите раздел "Информация о сервере" или "SSH доступ"
3. Найдите **IP адрес сервера** (например: `123.45.67.89`)
4. Используйте IP для подключения:

```bash
ssh dsc23ytp@123.45.67.89
```

---

## РЕШЕНИЕ 2: ИСПОЛЬЗОВАТЬ SSH АДРЕС BEGET

Beget обычно предоставляет специальный SSH адрес. Проверьте в панели:

- Может быть: `ssh.beget.com`
- Или: `dragon.beget.com`
- Или: `dsc23ytp@ssh.beget.com`

---

## РЕШЕНИЕ 3: УЗНАТЬ АДРЕС НА САМОМ СЕРВЕРЕ

Если вы уже подключены к серверу (через панель Beget или другой способ), выполните:

```bash
# Узнать IP адрес сервера
hostname -I

# Или
ip addr show

# Или
ifconfig
```

Запишите IP адрес и используйте его для подключения.

---

## РЕШЕНИЕ 4: ИСПОЛЬЗОВАТЬ VS CODE REMOTE SSH

VS Code Remote SSH может работать с локальными именами хостов через SSH config.

1. Установите расширение "Remote - SSH"
2. Настройте SSH config (см. ниже)
3. Подключайтесь через VS Code

---

## НАСТРОЙКА SSH CONFIG

Создайте файл: `C:\Users\ВашеИмя\.ssh\config`

Добавьте (замените IP_АДРЕС на реальный):

```
Host dragon
    HostName IP_АДРЕС_СЕРВЕРА
    User dsc23ytp
    IdentityFile C:\Users\ВашеИмя\.ssh\id_rsa
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

После этого можно подключаться: `ssh dragon`

---

## БЫСТРЫЙ СПОСОБ

Если вы уже подключены к серверу через панель Beget:

1. Откройте терминал в панели Beget
2. Выполните: `hostname -I` или `ip addr`
3. Запишите IP адрес
4. Используйте его для SSH подключения

---

## ПРОВЕРКА

После настройки попробуйте:

```bash
ssh dsc23ytp@IP_АДРЕС
```

Если подключилось - отлично! ✅

---

## АЛЬТЕРНАТИВА: ИСПОЛЬЗОВАТЬ ПАНЕЛЬ BEGET

Если SSH не работает, можно:
1. Использовать терминал в панели Beget
2. Загружать файлы через FTP/SFTP
3. Использовать File Manager в панели

Но SSH удобнее для разработки.
