# Отладка краша приложения в Android Studio

## 1. Подключение телефона

1. На телефоне: **Настройки** → **О телефоне** → 7 раз нажать на **Номер сборки**
2. Вернуться → **Для разработчиков** → включить **Отладка по USB**
3. Подключить телефон к компьютеру USB-кабелем
4. На телефоне подтвердить «Разрешить отладку по USB»

---

## 2. Открыть проект в Android Studio

1. Запустить **Android Studio**
2. **File** → **Open**
3. Выбрать: `c:\OSPanel\domains\Messager\mobile\android`
4. Дождаться синхронизации Gradle

---

## 3. Запуск в режиме отладки (Debug)

1. Вверху выберите **подключённый телефон** как устройство
2. Убедитесь, что выбран **debug** (не release)
3. Нажмите **Debug** (жук) или **Run** (зелёный треугольник)
4. Приложение установится и запустится

**Если Debug запускается** — проблема скорее в ProGuard (release). ProGuard уже отключён в build.gradle.

---

## 4. Смотреть логи (Logcat)

1. Внизу откройте вкладку **Logcat**
2. В фильтре выберите **Error** или введите: `AndroidRuntime`
3. Запустите приложение и смотрите красные строки при краше

**Или через командную строку:**
```powershell
adb logcat *:E
```
Запустите приложение и смотрите вывод.

---

## 5. Частые причины краша

| Ошибка в логах | Решение |
|----------------|---------|
| `SoLoader` / нативные библиотеки | Проверить Hermes, NDK |
| `Firebase` / google-services | Проверить google-services.json |
| `ReactNative` / JS bundle | Проверить Metro, bundle |
| `MainActivity` / ClassNotFoundException | Проверить наличие MainActivity.kt |
| ProGuard obfuscation | Отключить minifyEnabled для теста |
